Expressive CSS

Stop defining the same CSS properties over and over again. Use a core group of utility classes to keep your CSS lightweight, scalable and easy to understand. Document your classes as you go, such as in this starter example.

Base

Smart Defaults

html

Set default font styles, including family, color, size and line height. (See typography)

html {
    font-family: Verdana,sans-serif;
    color: #222;
    font-size: 1em;
    line-height: 1.4;
}

Utilities

Utility classes are at the heart of Expressive CSS. They are easy to write and understand. Using them reduces the necessity to re-declare the same sets of related properties over and over again, keeping your stylesheets more DRY and less complex.

Typography

Set smart defaults in your base classes for text size and styling (e.g. .bold .h2 .italic).

h1 h2 h3 h4 h5 h6 p

To keep things DRY and consistent across your site, include utility classes for each level of font-sizing. Set margin and padding to 0 for h1-h6.

h1,h2,h3,h4,h5,h6,p { padding:0; margin:0; }
h1, .h1 { font-size: 5em; }
h2, .h2 { font-size: 2.5em; }
h3, .h3 { font-size: 1.75em; }
h4, .h4 { font-size: 1.375em; }
h5, .h5 { font-size: 1.2em; }
h6, .h6 { font-size: 1em; }
p, .text { 
    padding-bottom:.5em;
    font-size: 1em;
    font-weight: normal;
}

Components

To reduce complexity in your CSS, rely on utility classes whenever possible. For elements that necessitate additional custom styling, use component classes. For these components, you may wish to use a CSS methodology such as OOCSS or BEM.

Below are some common examples of components.

.btn
button, .btn {
    display: inline-block;
    background: #9fc74d;
    display: inline;
    color: #fff;
    padding: 4px 6px;
    border-radius: 4px;
    transition: all .15s ease-in-out;
    border: none;
    outline: none;
    text-decoration: none;
    margin-bottom: 4px;
}
button:hover, .btn:hover {
    background: #5bb12f;
}

Other Components

Some components are truly one-offs. When you truly will not ever need to use that style anywhere else, such as in a graphic with absolutely positioned interior elements, use a very specifically named component class (.e.g. .animated-flying-tomato).
